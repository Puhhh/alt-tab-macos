# AltTabWindows

`AltTabWindows` — нативная macOS-утилита, которая переключает не приложения, а отдельные окна по `Option + Tab`.

Приложение показывает HUD-переключатель поверх всех пространств, собирает список видимых окон через `CGWindowList`, а затем переводит фокус на выбранное окно через Accessibility API.

## Возможности

- глобальный хоткей `Option + Tab`
- переключение между отдельными окнами, а не только между приложениями
- HUD-оверлей со списком окон и текущим выбором
- активация окна после отпускания `Option`
- отмена переключения по `Esc`
- иконка в строке меню и отдельное окно с состоянием разрешений
- работа на нескольких дисплеях с позиционированием HUD на релевантном экране

## Требования

- macOS 14.0+
- установленный Xcode
- выданный доступ в `System Settings > Privacy & Security > Accessibility`

В проекте нет внешних зависимостей: используется стандартный стек macOS API.

## Запуск из Xcode

1. Откройте `AltTabWindows.xcodeproj`.
2. Выберите схему `AltTabWindows`.
3. Запустите приложение.
4. После первого старта выдайте приложению Accessibility-доступ.

После запуска приложение работает как `LSUIElement`-утилита: остаётся в строке меню и не показывает иконку в Dock.

## Сборка из терминала

```bash
xcodebuild -project AltTabWindows.xcodeproj \
  -scheme AltTabWindows \
  -configuration Debug \
  -derivedDataPath .build \
  build
```

Собранное приложение появится по пути:

```text
.build/Build/Products/Debug/AltTabWindows.app
```

CLI-сборка проверена для текущего проекта и успешно складывает артефакты в локальную директорию `.build`.

## Разрешения

Без Accessibility-доступа приложение не сможет:

- читать заголовки и геометрию окон
- определять текущее активное окно
- переводить фокус на выбранное окно

Если доступ ещё не выдан, главное окно приложения покажет статус разрешения и кнопку для открытия системных настроек.

## Как пользоваться

1. Оставьте `AltTabWindows` запущенным в фоне.
2. Удерживайте `Option`.
3. Нажимайте `Tab`, чтобы идти вперёд по списку видимых окон.
4. Отпустите `Option`, чтобы активировать выбранное окно.
5. Нажмите `Esc`, чтобы закрыть HUD без переключения.

## Как устроено переключение

- список окон строится из видимых on-screen окон через `CGWindowListCopyWindowInfo`
- служебные окна системы отфильтровываются
- минимизированные окна исключаются
- порядок опирается на текущий z-order и локальную историю последних переключений
- при активации приложение пытается восстановить окно, поднять его и сделать главным через Accessibility API

Это даёт поведение, близкое к `Alt-Tab` для окон, но без внедрения приватных API.

## Ограничения

- приложение работает только с окнами, доступными через публичные macOS API
- без Accessibility-разрешения переключение недоступно
- минимизированные окна не попадают в список
- порядок не является точной копией системного MRU для всех сценариев
- качество сопоставления окна зависит от доступности AX-данных, заголовка и геометрии окна

## Структура проекта

Основные части приложения находятся в `Sources/AltTabWindows`:

- `AppController` координирует hotkey, сбор окон, историю и HUD
- `HotKeyCenter` регистрирует глобальный `Option + Tab`
- `WindowCatalog` собирает и сопоставляет видимые окна
- `AccessibilityBridge` читает и активирует окна через AX API
- `SwitcherPanelController` и `SwitcherPanelView` показывают HUD-переключатель
- `ContentView` показывает главное окно со статусом доступа и инструкциями

## Статус

Проект уже умеет переключать реальные окна и собираться в `.build`, но всё ещё остаётся компактной utility/MVP-реализацией, которую можно дальше улучшать по точности порядка окон, обработке edge cases и UX HUD.
